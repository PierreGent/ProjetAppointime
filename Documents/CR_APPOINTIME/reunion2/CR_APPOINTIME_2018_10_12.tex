\section{Description générale de l'application}
Nous nous sommes entendu sur la description générale de l'application
pour partir sur une bonne base commune. Nous avons convenu que
l'application sera découpée en deux grande parties :
\begin{itemize}
\item La partie client : 
  \begin{itemize}
    \item L'utilisateur peut rechercher des professionnels (Coiffeur,
      Mécanicien, plombier ...)
    \item Il peux ensuite prendre des rendez vous directement sur
        l'application selon le calandrier (disponibilité) du professionnel.
  \end{itemize}
\item La partie professionnel :
  \begin{itemize}
    \item Chaque professionnel a un calandrier sur lequel il a ajouté
      ses disponibilitées.
    \item Il voit en temps réel les rendez-vous pris par les client
        sur ce calendrier.
  \end{itemize}
\end{itemize}

\section{Propositions de fonctionalités}
\begin{itemize}
  \item Définition des types de rendez-vous par le professionnel :

    Exemple : Un coiffeur propose deux services, soit une coupe homme
    soit une coupe femme. Ces deux services prennent un temps
    différent et ne valent pas le même prix. Le professionnel va donc
    pouvoir paramétrer plusieurs types de rendez-vous (temps, prix
    ...).
  \item Gestion de proximité pour le client, l'application lui
    propose les professionnels du plus proche au moins proche.
  \item Système de paiement via l'application.
  \item Alerte au client pour lui rappeler son rendez-vous.
  \item Système de flash code qui permet de récuperer les informations
    d'un professionnel directement dans la rue.
   \item Système de navigation pour mener le client à son rendez-vous
     grâce au GPS.
   \item Système de confirmation de compte via numéro de téléphone.
   \item Si un client ne viens pas à un rendez-vous, le professionnel
     peut le signaler et faire perdre une étoile au client. Le client
     ayant trois étoile à son inscription, il sera banni si sont compte
     atteint zéro étoiles. Le client récupere 1 étoile tout les X
     rendez-vous.
   \item Notification quand un nouveau professionnel commence à
     utiliser l'application proche d'un client. 
\end{itemize}
